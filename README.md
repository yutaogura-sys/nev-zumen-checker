# NeV 図面チェックツール（統合版）

NeV（次世代自動車振興センター）充電インフラ補助金の**交付申請図面4種**の要件適合を、
1つのツールで種別タブを切り替えてチェックするツール。

- 対象図面: 設置場所見取図 / 平面図 / 配線ルート図 / 電気系統図
- 対象年度: **R7補正**（令和7年度補正予算）。要件の根拠は `../R7補正_NeV要件/` を参照。
- 技術: 静的Webアプリ（ビルド不要・バックエンドなし）＋ Gemini API（BYOK＝利用者が自分のAPIキーを入力）
- 前身: 図面種別ごとに作られた4つの個別ツールを統合。設計は `../統合ツール設計案_v1.0.md`。

> 本ツールはAIによる一次チェック補助です。**最終判断は必ず人間が目視で行ってください。**

## ディレクトリ構成

```
nev-drawing-checker/
├─ index.html            … 1ページ・4タブ（P2で実装）
├─ core/                 … 4図面共通ライブラリ（図面種別に依存しない）
│   ├─ util.js           … 文字正規化・HTMLエスケープ・ファイルサイズ整形・OS判定
│   ├─ pdf.js            … PDF→画像化 / PDFネイティブ入力 / プレビュー生成（pdf.js依存）
│   ├─ gemini.js         … Gemini API 呼び出し・リトライ・モデル検証・JSON抽出
│   ├─ cost.js           … 料金概算 ＋ 料金上限の警告・超過表示（ブラウザ単位で累計）
│   ├─ aggregate.js      … チェック結果の集計・総合判定（定義配列を渡して汎用化, na対応）
│   ├─ prompt.js         … 単一定義(rules)から Gemini プロンプトを自動生成（二重管理の解消）
│   ├─ deterministic.js  … 決定論的クロスチェック（AI抽出値をコードで再検証: 幅2.5m/デマンド定格台数）
│   ├─ crosscheck.js     … 案件まとめ: 図面間の整合性（施設名・作成者・充電台数の一致）
│   ├─ vote.js           … 自己一致（高精度モード: N回判定の多数決、割れはwarn）
│   ├─ reconcile.js      … 配線数値の三者照合（統括表⇔旗上げ⇔寸法、桁違い検出）
│   └─ rules-registry.js … 図面種別ごとの要件定義（rules/）の登録・取得
├─ rules/                … 図面種別ごとの「単一定義」（P2で実装）
│   ├─ mitori.js / heimen.js / haisen.js / keitou.js
├─ css/                  … スタイル（配線ルート図ツールの見た目をベース）
├─ app.js                … 個別チェックのオーケストレーション（index.html）
├─ batch.html / batch.js … 案件まとめモード（4図面一括＋図面間整合性チェック）
└─ index.html            … 個別チェック（4図面タブ）
```

## 設計の要点

- **単一定義（rules/）**: 「何をチェックするか」の唯一の情報源。プロンプトはこの定義から
  自動生成する（P3）。項目定義とプロンプト本文を別々に手直しする二重管理を構造的に排除する。
- **core は図面種別に依存しない**: PDF処理・API呼び出し・料金・集計は共通。
  図面ごとの違いは `rules/` の定義と（P3以降の）プロンプト生成のみ。
- **料金上限の警告**（組織方針・必須）: `core/cost.js` の `CapTracker` がブラウザ単位で
  API利用額を累計し、上限の一定割合で警告、超過時は「現在 ○○円 オーバー」を明示する。

## core の依存関係

| モジュール | ブラウザ依存 | Nodeで単体テスト可 |
|---|---|---|
| util.js | なし | ○ |
| cost.js | localStorage（任意） | ○（CapTrackerはストア注入で可） |
| aggregate.js | なし | ○ |
| pdf.js | pdf.js / document / canvas | ×（ブラウザ専用） |
| gemini.js | fetch（Node18+可） | 一部○ |

## 起動方法

静的ファイルなのでローカルサーバーで開くだけ（ビルド不要）:

```
python -m http.server 8777 --directory nev-drawing-checker
# ブラウザで http://localhost:8777/ を開く
```

APIキー（Gemini, BYOK）を入力 → 図面種別タブを選択 → 事業区分（基礎/目的地）→ PDFをアップロード → 実行。

## テスト

Node で実行（ブラウザ不要）:

```
node tests/test_core.js    # core モジュール（util/aggregate/cost）の単体テスト

node tests/test_deterministic.js  # 決定論チェック（幅2.5m/デマンド定格台数）
node tests/test_crosscheck.js     # 図面間整合性（施設名・作成者・台数）
```

## サポート・利用条件（配布先向け）
- **費用**: 本ツールはBYOK方式です。API利用料は利用者自身のGoogleアカウントに発生します（Gemini 2.5 Flashに無料枠あり。無料枠は1日あたりの回数制限があり、超えると翌日まで待つか課金設定が必要です）。画面内に料金概算・月次上限の警告機能があります。
- **無保証**: 判定結果はAIによる一次チェックです。最終判断は必ず要領・記入例の原本に基づき目視で行ってください（詳細: LICENSE.txt）。
- **問い合わせ・不具合報告**: 本リポジトリの [Issues](../../issues) へお願いします。
- **プロンプトインジェクション対策**: 図面内の文字列を「指示」として扱わないようAIに明示しています（審査を誘導する不審な文言は「要確認」として報告されます）。ただし完全な防御は保証できないため、判定結果は必ず目視確認してください。
